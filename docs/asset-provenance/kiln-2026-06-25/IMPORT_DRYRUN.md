## Rig wiring — RESOLVED via a canonical joint contract (2026-06-25)
The fresh Kiln assets are **richly rigged** — far better than the old rig-deleted
pixel-forge batch — but ship **internally-inconsistent, generator-native joint
names** (Kiln emits `Joint_Rotor` on the Huey but `Joint_MainRotor` on the OH-6;
`Joint_PropLeft` on the AC-47 but `Joint_PropL` on the OV-10; `Joint_Gun` on the M48
where the consumer wants `Joint_MainGun`). The old `rig-grafts.json` carried a
hand-authored rename **per asset**, reverse-engineered from the previous batch — a
treadmill that broke on every re-roll.

**Principled fix (no patchwork):** the rename treadmill is replaced by a single
**canonical rig contract**, `scripts/asset-import/joint-taxonomy.json`. The importer
now auto-detects every `Joint_*` node, maps its generator-native name to ONE canonical
name + semantic role via the taxonomy's alias table, renames the GLB node, records
`{name,type,spinAxis}`, geometrically cross-checks each spin axis, and **validates that
every articulated asset resolved the joints its rig profile requires** (a missing
rotor/turret now fails the import instead of dying silently at runtime). A future batch
that renames a part again is a one-line `aliases` edit, not a per-asset graft.
`rig-grafts.json` is deleted.

Canonical vocabulary established (alias → canonical):
| role | canonical | Kiln aliases normalized | type / spinAxis |
|---|---|---|---|
| main rotor | `Joint_MainRotor` | `Joint_Rotor`, `Joint_MainRotor` | mainBlades / y |
| tail rotor | `Joint_TailRotor` | `Joint_TailRotor` | tailBlades / z |
| tandem rotors | `Joint_FrontRotor`/`Joint_RearRotor` | same | mainBlades / y |
| twin prop | `Joint_PropellerL`/`Joint_PropellerR` | `Joint_PropLeft/Right`, `Joint_PropL/R` | mainBlades / x |
| single prop | `Joint_Propeller` | `Joint_Propeller` | mainBlades / x |
| tank turret | `Joint_Turret` | `Joint_Turret` | — |
| tank main gun | `Joint_MainGun` | `Joint_Gun`, `Joint_MainGun` | — |

Weapon magazine/muzzle anchors are now classified by regex rules (include/exclude over
`Mesh_*` names) instead of hand lists — magazine = the detachable body (well + release
catch excluded), muzzle = the forward device (bare barrel omitted; runtime falls back
to `Mesh_Barrel`), with per-slug overrides for the M79/RPG-7.

**Dry-run rig result on the Kiln batch: 0 errors, 0 warnings, 14 info.** Every
helicopter (UH-1, UH-1C, AH-1, OH-6, HH-3E), tandem (CH-47), prop aircraft
(A-1, AC-47, OV-10), and tank (M48, T-54) resolves its canonical joints; zero spin-axis
disagreements (geometric thin-axis confirms main=y/tail=z/prop=x). The 14 info lines
record **un-wired articulation the Kiln assets expose for future cycles** — e.g. M113
cupola + gun elevation + ramps, Ontos turret-yaw/gun-pitch, A-37 gear + bomb/rocket
pods, PBR/Swift-Boat gun turrets + mortar mounts, C-130's four props, PT-76 sprocket/
idler pivots. Wiring those is a follow-up (add the slug to `assetRigProfiles` + a role).

Contract is verified: `tsconfig.scripts.json` typecheck clean;
`HelicopterGeometry`/`FixedWingConfigs`/`WeaponRig*` tests (87) green — those tests ARE
the consumer contract (`Joint_PropellerR/L` + spinAxis `x`; helis declare main+tail),
and the canonicalization is what makes the Kiln raw names satisfy them.

Enter/seat and chase-camera remain config-driven (vehicle adapter configs, not GLB
nodes) → unaffected. The M48 GLB turret swap (`VehicleGlbVisuals`) needs exact
`Joint_Turret` + `Joint_MainGun` node names — both now produced by canonicalization.
