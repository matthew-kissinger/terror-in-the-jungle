## Resuming a halted campaign

If a hard-stop fires:

1. The orchestrator updates this file: failed cycle status → `BLOCKED <one-line cause>`.
2. `docs/AGENT_ORCHESTRATION.md` "Current cycle" stays pointing at the failed cycle.
3. The human resolves the underlying issue (e.g. fence change discussed +
   approved, perf regression diagnosed + fixed).
4. To resume: clear the BLOCKED status in this file, run `/orchestrate`
   again. The orchestrator reads the campaign and continues from the same
   cycle.

## Manual override

To skip a cycle (e.g. it's been overtaken by events): mark its status
`skipped` in this file with a one-line cause. The orchestrator then
advances past it.

To insert a new cycle mid-campaign: edit this file, add the cycle row in
the right position. Make sure the cycle's brief exists at the named path
before running `/orchestrate`.
