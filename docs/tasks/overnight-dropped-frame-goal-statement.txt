Aligned finish-line goal (2026-06-15):

Reduce player-visible dropped-frame time and stutter in real Open Frontier and
A Shau gameplay while preserving the current game experience. Success is not a
lower metric produced by deleting, hiding, delaying, or weakening the game:
combat pressure, terrain and vegetation readability, wildlife/animals where
enabled, war-asset visuals, weather/atmosphere, draw distance, player flow, and
mode startup/deploy behavior must remain materially equivalent or better.

Completion evidence required:
1. The perf harness measures the player-visible symptom directly: dropped-frame
   time, rAF gaps, long tasks / LoAFs, frame-time tails, and presentation epochs.
   It must also expose enough context to distrust itself when data looks wrong:
   camera/recoil/firing state, driver aim/slew/retarget state, terrain/CDLOD
   selection and render-sync epochs, LOS/fire-authority decisions, render
   submission categories, wildlife/world-feature/vehicle/NPC representation
   pressure, browser stall entries, and asset/texture residency stages.
2. Quiet-machine seeded A Shau and Open Frontier captures show the dropped-frame
   gate passing with real combat and representative world content. Static tests,
   typecheck, lint, build, and local code reasoning are necessary but never
   sufficient to mark this goal complete.
3. If metrics do not improve, or if a run contradicts the owner-visible
   experience, treat that as a finding. First check for harness blind spots,
   missing systems, dirty-machine noise, synthetic-driver behavior, simulation
   shortcuts, terrain/camera epoch mismatch, missing attribution, or
   unmeasured asset/texture upload work. Do not force an optimization story.
4. Every performance change must name what it preserves. Same-experience
   optimizations are allowed: better scheduling, batching, upload ranges,
   resource reuse, texture/asset pipeline fixes, representation leases,
   instrumentation, and authority-chain corrections. Content cuts, narrower
   draw distance, reduced animals, weaker combat, less terrain, less vegetation,
   or degraded visuals are not valid wins unless the owner explicitly changes
   the game design.
5. The owner-reported terrain/camera glitch remains a first-class symptom until
   disproven. "Clipping," "underside," and "white gap" are imprecise symptom
   language, not a settled diagnosis. Keep camera transform/projection cadence,
   recoil/aim adjustment, harness interpolation/slew, CDLOD selection,
   terrain streaming/culling epochs, terrain skirts/backfaces, render timing,
   and shooting-path work in the suspect set until artifacts narrow it.

Latest static checkpoint:
- CDLOD terrain integrity checkpoint:
  Reference CDLOD codebases point to one invariant that matters for the owner
  symptom: selected tile bounds, transform, LOD/morph params, and heightmap
  sampling coordinates must be submitted coherently. Our Three/WebGPU terrain
  splits that across `instanceMatrix`, `tileParams0`, and `tileParams1`, so
  partial uploads are a custom footgun. `CDLODRenderer` now rewrites the full
  active terrain instance prefix and marks all terrain instance buffers dirty
  together on each submission. `?terrainForceInstanceUpload=1` is available in
  dev/perf-harness runs to bypass unchanged-tile submission skips and test
  whether a bad GPU terrain-buffer state was being preserved. This preserves
  terrain content, draw distance, morph rules, vegetation, wildlife, war
  assets, atmosphere, combat pressure, and player flow. It is not runtime proof
  yet; the next A Shau capture must compare normal vs forced terrain instance
  upload and then isolate low-sun occlusion / far-canopy tint if dark bands
  remain.
- Shot epochs now include sanitized presentation context from the engine's
  rendered-camera epoch recorder: frame count, camera epochs, terrain tile
  hashes, terrain clearance/area-readiness, CDLOD render-sync state, and
  renderer stats.
- `perf-capture` now warns via
  `harness_shot_presentation_context_equivalence` when firing windows lack
  presentation context or show large intra-frame camera deltas, low camera
  terrain clearance, terrain tile-hash churn, or not-ready terrain. This is
  diagnostic only; it does not prove dropped-frame time improved.
