## 2. Redundancy (same information in multiple files)

### Agent Onboarding Triplication

`AGENTS.md`, `CLAUDE.md`, and `docs/README.md` all serve the same purpose: orient an AI agent to the codebase. They overlap heavily:

- **Commands**: listed in `AGENTS.md`, `CLAUDE.md`, and `README.md` (3 copies)
- **Architecture overview**: `AGENTS.md` and `CLAUDE.md` both describe entry points, key systems, runtime landmarks
- **Doc read order**: `AGENTS.md` and `docs/README.md` both specify read order (slightly different orderings)
- **Documentation contract / update rules**: all three files state update rules

`CLAUDE.md` is essentially `AGENTS.md` + a massive append-only changelog (items 1–25). ~70% of its content is historical session notes that duplicate information better tracked in `PLAN_STATE.md` and `NEXT_WORK.md`.

### Tracking Overlap

`PLAN_STATE.md`, `NEXT_WORK.md`, and `ARCHITECTURE_RECOVERY_PLAN.md` overlap significantly:

- Wave 3 items in `PLAN_STATE` duplicate architecture debt in `ARCHITECTURE_RECOVERY_PLAN`
- Both track "Combat AI p99 ~35ms" as a known issue
- Both have "Known Bugs / Active Defects" and "Open Risks" sections covering the same ground
- `NEXT_WORK.md` completion log repeats what `PLAN_STATE` waves already record

### Stats Duplication

The same metrics (test count, file count, system count, CSS module count) appear in 3+ files. When they change, only 1 or 2 files get updated, making every doc partially wrong at any given time.

## 5. Structural Problems

### No Single Source of Truth

The same fact (e.g., "3,591 tests passing") appears in 3+ files. When tests change, only 1 or 2 files get updated. This makes every doc partially wrong at any given time.

### Append-Only Culture

`CLAUDE.md`, `ARCHITECTURE_RECOVERY_PLAN.md`, and `PERF_FRONTIER.md` all grow by appending. Nobody removes stale entries. The Keep Decisions list is the worst offender — 166 items where maybe 20 are architecturally significant decisions and the rest are "we deleted a dead file."

### Ephemeral State in Committed Files

`progress.md` and parts of `CLAUDE.md` track per-session work that belongs in git commit messages or session-local notes, not committed documentation.

### Agent Docs vs Human Docs Not Separated

`AGENTS.md` and `CLAUDE.md` exist for AI agents. `ROADMAP.md` and `README.md` exist for humans. But they overlap in content and neither audience gets a clean, minimal surface.
