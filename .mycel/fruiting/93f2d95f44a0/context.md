# Task: Improve NPC Death Animations

## Context
Currently when NPCs die, the death is just a sprite swap (the billboard texture changes). This lacks visual impact and makes kills feel unsatisfying. The GDD explicitly mentions "Death animations are just sprite swaps - need proper death animations" under Combat Feel & Feedback.

## Discovery - Read These Files First
1. src/systems/combat/CombatantRenderer.ts - How NPCs are rendered
2. src/systems/combat/CombatantCombat.ts - Where death is handled
3. src/types/Combatant.ts - Combatant data structure
4. src/systems/world/billboard/BillboardInstanceManager.ts - Billboard rendering

## Memory Context (from repo conventions)
- Death animations use isDying flag and deathProgress (0-1) on Combatant
- Renderer interprets deathProgress to drive visual effects
- The pattern is already established, just needs implementation

## Requirements
1. **Ragdoll-style fall**: When NPC dies, animate falling backward/forward
   - Direction based on hit direction (shot from front = fall back)
   - Duration: 0.6-0.8 seconds
   - Include rotation (tipping over)

2. **Ground impact**: When hitting ground, slight bounce/settle
   - Final position slightly offset from death point
   - Body stays on ground (don't despawn immediately)

3. **Fade out**: After 3-5 seconds on ground, fade out
   - Opacity transition over 1 second
   - Then remove from scene

## Implementation Steps
1. Add to Combatant type (if not present):
   - isDying: boolean
   - deathProgress: number (0-1)
   - deathDirection: Vector3 (direction of killing shot)
   - deathStartTime: number

2. In CombatantCombat.ts - onDeath:
   - Set isDying = true
   - Capture death direction from damage source
   - Start death animation timer

3. In CombatantRenderer.ts - update loop:
   - If isDying, interpolate deathProgress (0 to 1 over 0.7s)
   - Apply rotation based on deathProgress (tip 0 to 90 degrees)
   - Apply Y position (arc down to ground)
   - At progress 1.0, start fadeout timer

4. Billboard modifications:
   - Support opacity/alpha for fadeout
   - Support rotation for falling

## Animation Curve
- Use easeOutQuad for natural fall acceleration
- Position: start Y -> ground Y (parabolic arc)
- Rotation: 0 -> 75-90 degrees (based on direction)

## Validation
- Kill an NPC from the front -> falls backward
- Kill an NPC from behind -> falls forward
- Death takes ~0.7 seconds to complete
- Body stays on ground for 3-5 seconds
- Body fades out smoothly
- No visual glitches during animation
